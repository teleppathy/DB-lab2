# Опис схеми бази даних
Ця база даних призначена для створення платформи типу нетфлікс та київстар ТБ. Схема забезпечує цілісність даних завдяки використанню первинних та зовнішніх ключів, а також спеціальних типів ENUM для стандартизації значень.

# Перелік таблиць та опис структури:
customer (Клієнти):

## customer_id: Первинний ключ (SERIAL).

- email: Унікальне поле (UNIQUE).

- Обмеження: Пароль повинен бути не коротшим за 6 символів (CHECK length >= 6).

## subscription (Підписки):

- subscription_id: Первинний ключ.

- customer_id: Зовнішній ключ, що посилається на customer(customer_id).

- type: Тип підписки (ENUM: сімейна, студентська, стандартна).

## payment (Платежі):

- customer_id та subscription_id: Зовнішні ключі.

- amount: Сума платежу (обмеження CHECK >= 0).

- studio (Кіностудії):

- studio_id: Первинний ключ.

- name: Унікальна назва студії.

## film (Фільми):

- studio_id: Зовнішній ключ, що посилається на studio(studio_id).

- age_restriction: Вікове обмеження (ENUM: 0+, 12+, 16+, 18+, 21+).

- actor та director (Актори та Режисери): Містять персональні дані (прізвище, ім'я, країна).

## Таблиці зв'язків (Many-to-Many):

- film_genre, film_actor, film_director: Використовують складені первинні ключі для зв'язку фільмів з відповідними категоріями та людьми.
